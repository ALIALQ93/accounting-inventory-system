# 🛡️ دليل حذف العملاء والموردين مع المعاملات المحاسبية

## ❌ الإجابة: لا، لا يتم حذف الحساب إذا كان يحتوي على عمليات

---

## 📊 آلية التحقق من المعاملات

### 1️⃣ ✅ فحص العمليات المرتبطة بالعميل/المورد

قبل حذف العميل/المورد، النظام يتحقق من:

```javascript
async checkPartyTransactions(party) {
    // ✅ فحص فواتير المبيعات
    sales.where('customerId', '==', party.id)
    
    // ✅ فحص فواتير المشتريات
    purchases.where('supplierId', '==', party.id)
    
    // ✅ فحص إيصالات وسندات
    receipts.where('partyId', '==', party.id)
    
    // ✅ فحص القيود المحاسبية
    journalEntries.where('partyId', '==', party.id)
    
    // ✅ فحص حركات الخزينة
    treasuryTransactions.where('partyId', '==', party.id)
}
```

**النتيجة**: إذا وُجدت أي عمليات، يتم منع الحذف وإظهار رسالة خطأ.

---

### 2️⃣ ✅ فحص المعاملات المالية على الحساب

بعد التأكد من عدم وجود عمليات مرتبطة، النظام يتحقق من الحساب المحاسبي:

```javascript
async checkAccountTransactions(accountId) {
    // ✅ فحص الرصيد
    if (balance !== 0) return true;
    
    // ✅ فحص الحركة المدينة
    if (debit !== 0) return true;
    
    // ✅ فحص الحركة الدائنة
    if (credit !== 0) return true;
    
    // ✅ فحص المعاملات في جدول accountTransactions
    accountTransactions.where('accountId', '==', accountId)
    
    // ✅ فحص القيود في جدول journalEntries
    journalEntries.where('accounts', 'array-contains', accountId)
}
```

**النتيجة**: إذا كانت أي قيمة غير صفر أو وُجدت معاملات، يتم منع الحذف.

## 🚫 رسائل الخطأ المعروضة

### رسالة 1: وجود عمليات مرتبطة
```
❌ لا يمكن حذف هذا العميل/المورد!

🔗 السبب: يوجد عمليات مرتبطة بهذا العميل/المورد

📊 العمليات الموجودة:
• فواتير مبيعات (X عنصر)
• قيود محاسبية (Y عنصر)
• ...

💡 للحذف يجب أولاً:
1. حذف أو تعديل جميع العمليات المرتبطة
2. التأكد من عدم وجود معاملات مالية
3. ثم إعادة المحاولة
```

### رسالة 2: وجود معاملات على الحساب
```
❌ لا يمكن حذف هذا العميل/المورد!

💰 السبب: يوجد معاملات مالية على الحساب المرتبط

🏦 الحساب المحاسبي المرتبط به معاملات مالية
لا يمكن حذف العميل/المورد إذا كان حسابه المحاسبي يحتوي على معاملات

💡 للحذف يجب أولاً:
1. مراجعة الحساب المحاسبي في شجرة الحسابات
2. نقل أو حذف جميع المعاملات المالية
3. التأكد من أن رصيد الحساب = صفر
4. ثم إعادة المحاولة
```

---

## ✅ متى يتم الحذف؟

يتم حذف العميل/المورد **فقط** عندما:

1. ✅ **لا توجد عمليات مرتبطة** في:
   - فواتير المبيعات
   - فواتير المشتريات
   - إيصالات وسندات
   - قيود محاسبية
   - حركات خزينة

2. ✅ **الحساب المحاسبي نظيف**:
   - الرصيد = صفر (`balance === 0`)
   - الحركة المدينة = صفر (`debit === 0`)
   - الحركة الدائنة = صفر (`credit === 0`)
   - لا توجد معاملات في `accountTransactions`
   - لا توجد قيود في `journalEntries`

---

## 🔄 عملية الحذف (عند توفر الشروط)

```javascript
// 1. ✅ فحص العمليات - إذا وُجدت، إيقاف العملية
if (hasPartyTransactions) {
    return; // منع الحذف
}

// 2. ✅ فحص المعاملات على الحساب - إذا وُجدت، إيقاف العملية
if (hasAccountTransactions) {
    return; // منع الحذف
}

// 3. ✅ إذا كان كل شيء نظيفاً:
// - حذف الحساب الفرعي من chartOfAccounts
await deleteLinkedAccount(party.subAccountId);

// - حذف العميل/المورد من parties
await db.collection('parties').doc(partyId).delete();
```

## 💡 كيفية التعامل مع الحسابات التي تحتوي على عمليات

### الخيار 1: نقل المعاملات
1. إنشاء عميل/مورد جديد
2. نقل جميع المعاملات إليه
3. حذف العميل/المورد القديم

### الخيار 2: إغلاق الحساب
1. تعديل الحساب إلى "معطل" بدلاً من حذفه
2. الاحتفاظ بالبيانات التاريخية
3. منع إضافة معاملات جديدة

### الخيار 3: تصفية الحساب
1. تصفية جميع الأرصدة (تحويلها لحساب آخر)
2. التأكد من أن الرصيد = صفر
3. حذف العميل/المورد

## 📋 مثال عملي

### المثال 1: حساب به رصيد
```javascript
// بيانات الحساب:
{
    balance: 5000,     // ❌ رصيد غير صفر
    debit: 10000,
    credit: 5000
}

// النتيجة:
❌ لا يمكن حذف العميل - الحساب يحتوي على معاملات
```

### المثال 2: حساب نظيف
```javascript
// بيانات الحساب:
{
    balance: 0,        // ✅ رصيد صفر
    debit: 0,          // ✅ حركة مدينة صفر
    credit: 0          // ✅ حركة دائنة صفر
}

// لا توجد معاملات في:
// - accountTransactions
// - journalEntries

// النتيجة:
✅ يمكن حذف العميل والحساب
```

## 🎯 الخلاصة

| الحالة | هل يمكن الحذف؟ | السبب |
|--------|----------------|-------|
| حساب برصيد ≠ 0 | ❌ لا | الرصيد يجب أن يكون صفر |
| حساب بحركة مدينة ≠ 0 | ❌ لا | يجب تصفية جميع الحركات |
| حساب بحركة دائنة ≠ 0 | ❌ لا | يجب تصفية جميع الحركات |
| حساب بمعاملات في `accountTransactions` | ❌ لا | يجب حذف/نقل المعاملات |
| حساب بقيود في `journalEntries` | ❌ لا | يجب حذف/نقل القيود |
| **حساب نظيف تماماً** | ✅ **نعم** | **يمكن الحذف** |

---

**💡 نصيحة**: 
- ✅ **لا تحذف** العملاء/الموردين الذين لديهم معاملات محاسبية
- ✅ **استخدم** خيار "تعطيل" بدلاً من الحذف للاحتفاظ بالبيانات
- ✅ **نقّل** المعاملات لحساب آخر إذا أردت الحذف
