## 📋 ملخص التنفيذ

### ✅ النقاط الإيجابية:
1. **التحقق من المخزون**: يتم التحقق من توفر الكمية قبل الإرجاع
2. **التحقق من التوازن**: يتم التحقق من توازن القيد المحاسبي قبل الحفظ
3. **معالجة الأخطاء**: معالجة شاملة للأخطاء
4. **تتبع الحركات**: يتم تسجيل جميع حركات المخزن بشكل صحيح

## 🔍 تحليل القواعد المحاسبية

### 1. إضافة مرتجع مشتريات جديد

#### ✅ الترتيب الصحيح:
1. حفظ المرتجع أولاً (للحصول على ID)
2. تحديث المخزن (طرح الكميات)
3. توليد القيد المحاسبي

**الترتيب صحيح ✅**

---

### 2. القيود المحاسبية - القاعدة الأساسية

#### ✅ القاعدة المحاسبية لمرتجع المشتريات:
```
عكس قيد المشتريات الأصلي:
- المشتريات الأصلية: مدين المشتريات، دائن المورد
- المرتجع: مدين المورد، دائن المشتريات (عكس)
```

**القاعدة صحيحة ✅**

---

### 3. القيود المحاسبية - الدفع الآجل (Credit)

#### ✅ القيد الأساسي:
```
مدين: حساب المورد (total)
دائن: حساب المشتريات/المخزون (total)
```

**التحليل:**
- ✅ صحيح: عند إرجاع مشتريات آجلة، المورد مدين (نسترد منه)، والمشتريات دائن (نقلل المشتريات)

**القاعدة صحيحة ✅**

---

### 4. القيود المحاسبية - الدفع النقدي (Cash)

#### ⚠️ مشكلة محتملة في الكود الحالي:

**الكود الحالي:**
```javascript
// Credit: Purchases/Inventory account
const purchaseAccountId = settings.defaultCounterAccountId || null;
if (purchaseAccountId) {
    // دائن المشتريات فقط
    entryData.entries.push(purchaseEntry);
} else if (paymentMethod === 'cash') {
    // إذا لم يكن هناك purchaseAccountId: دائن النقدية فقط
    entryData.entries.push(cashEntry);
}
```

**المشكلة:**
- في حالة الدفع النقدي، يجب أن يكون هناك قيدان:
  1. مدين: النقدية (نسترد النقد)
  2. دائن: المشتريات (نقلل المشتريات)

**الكود الحالي:**
- إذا كان هناك `purchaseAccountId`: دائن المشتريات فقط ❌ (ناقص مدين النقدية)
- إذا لم يكن هناك `purchaseAccountId`: دائن النقدية فقط ❌ (ناقص دائن المشتريات)

**التصحيح المطلوب:**
```javascript
// في حالة الدفع النقدي:
// 1. دائن المشتريات (إذا كان هناك purchaseAccountId)
// 2. مدين النقدية (للاسترداد)
```

### 5. الخصومات والإضافات

#### ✅ القاعدة الحالية:

**الخصومات:**
```
دائن: حساب الخصم (إيراد)
مدين: الحساب المقابل (حساب المورد أو النقدية حسب طريقة الدفع)
```

**الإضافات:**
```
مدين: حساب الإضافة (مصروف)
دائن: الحساب المقابل (حساب المورد أو النقدية حسب طريقة الدفع)
```

**التحليل:**
- ✅ القاعدة صحيحة محاسبياً
- ✅ معالجة الخصومات والإضافات صحيحة

**القاعدة صحيحة ✅**

## 🔍 تحليل حركات المخزن

### 1. إضافة مرتجع جديد

#### ✅ القاعدة الحالية:
```javascript
- تحويل الكمية إلى الوحدة الأساسية
- طرح الكمية من المخزن (subtract)
- تسجيل حركة مخزن (type: 'out')
- التحقق من توفر الكمية قبل الطرح
```

**التحليل:**
- ✅ استخدام الوحدة الأساسية صحيح
- ✅ طرح الكمية من المخزن صحيح (عكس المشتريات)
- ✅ التحقق من توفر الكمية قبل الطرح صحيح
- ✅ تسجيل الحركة مع جميع التفاصيل صحيح

**القاعدة صحيحة ✅**

### 2. تعديل مرتجع

**التحليل:**
- ✅ الترتيب صحيح

## ⚠️ المشاكل والتوصيات

### 1. مشكلة حرجة: القيد المحاسبي في الدفع النقدي

#### المشكلة:
في حالة الدفع النقدي، الكود الحالي لا يضيف قيد مدين النقدية.

**الكود الحالي:**
```javascript
// Credit: Purchases/Inventory account
const purchaseAccountId = settings.defaultCounterAccountId || null;
if (purchaseAccountId) {
    // دائن المشتريات فقط
    entryData.entries.push(purchaseEntry);
} else if (paymentMethod === 'cash') {
    // دائن النقدية فقط
    entryData.entries.push(cashEntry);
}
```

**المشكلة:**
- إذا كان هناك `purchaseAccountId`: القيد غير متوازن (ناقص مدين النقدية)
- إذا لم يكن هناك `purchaseAccountId`: القيد غير متوازن (ناقص دائن المشتريات)

**التصحيح المطلوب:**
```javascript
// في حالة الدفع النقدي:
if (paymentMethod === 'cash') {
    // 1. دائن المشتريات (إذا كان هناك purchaseAccountId)
    if (purchaseAccountId) {
        entryData.entries.push(purchaseEntry);
    }
    
    // 2. مدين النقدية (للاسترداد)
    const cashAccountId = settings.defaultCreditAccount || null;
    if (cashAccountId) {
        const cashEntry = {
            accountId: cashAccountId,
            debit: totalInBaseCurrency,  // ✅ مدين (نسترد النقد)
            credit: 0,
            originalAmount: returnData.total || 0,
            originalCurrency: invoiceCurrency,
            description: `مرتجع شراء رقم ${returnData.invoiceNo} - نقدي`
        };
        entryData.entries.push(addAccountInfo(cashEntry, cashAccountId));
    }
}
```
