## 🎯 ما هي الوحدات الفرعية؟

### **المفهوم:**

```
الوحدات الفرعية تتيح بيع المنتج بوحدات مختلفة!

مثال:
  المنتج: بسكويت
  الوحدة الأساسية: علبة (حبة)
  
  الوحدات الفرعية:
    - 📦 صندوق صغير = 12 علبة
    - 📦 صندوق كبير = 24 علبة
    - 📦 كرتون = 6 صناديق كبيرة = 144 علبة
```

## ⚙️ كيفية الاستخدام

### **خطوة 1: إضافة منتج**

```
1. ✅ افتح "إضافة منتج"
2. ✅ اختر الوحدة الأساسية (مثلاً: "قطعة")
3. ✅ انقر "إضافة وحدة فرعية"
```

### **خطوة 2: تحديد الوحدة الفرعية**

```
1. ✅ اختر الوحدة من القائمة (مثلاً: "صندوق")
2. ✅ أدخل معامل التحويل (مثلاً: 12)
3. ✅ شاهد الملاحظة تظهر تلقائياً:
   
   ✅ 1 صندوق = 12 قطعة
   مثال: إذا اشتريت 12 قطعة، تحصل على 1 صندوق
```

### **خطوة 3: إضافة وحدات متعددة**

```
يمكنك إضافة أكثر من وحدة فرعية!

مثال:
  الوحدة الأساسية: قطعة
  
  وحدة فرعية #1:
    - الوحدة: صندوق صغير
    - المعامل: 12
    - الملاحظة: 1 صندوق صغير = 12 قطعة
  
  وحدة فرعية #2:
    - الوحدة: صندوق كبير
    - المعامل: 24
    - الملاحظة: 1 صندوق كبير = 24 قطعة
  
  وحدة فرعية #3:
    - الوحدة: كرتون
    - المعامل: 144
    - الملاحظة: 1 كرتون = 144 قطعة
```

## 🔍 الوظائف JavaScript

### **1. addSubUnit()**

```javascript
✅ يتحقق من اختيار الوحدة الأساسية
✅ يولد HTML للوحدة الفرعية
✅ يضيف event listeners
✅ يعرض رقم متسلسل (#1, #2, #3)
✅ يفلتر الوحدات (يخفي الوحدة الأساسية)
```

### **2. removeSubUnit(id)**

```javascript
✅ يحذف الوحدة الفرعية من DOM
✅ تأكيد سريع
```

### **3. updateConversionNote(id)**

```javascript
✅ يحسب التعادل
✅ يعرض ملاحظة واضحة:
   "1 صندوق = 12 قطعة"
✅ يضيف مثال توضيحي
✅ يتحدث فوراً عند تغيير القيم
```

### **4. collectSubUnitsData()**

```javascript
✅ يجمع بيانات جميع الوحدات الفرعية
✅ يرجع array من objects:
   [
     { unitId: "x", factor: 12 },
     { unitId: "y", factor: 24 }
   ]
✅ يُستدعى عند الحفظ
```

### **5. clearSubUnits()**

```javascript
✅ يمسح جميع الوحدات الفرعية
✅ يُستدعى عند:
   - فتح modal جديد
   - قبل تحميل بيانات التعديل
```

## 🔄 التدفق الكامل

### **عند الإضافة:**

```
1. المستخدم ينقر "إضافة منتج"
2. يختار الوحدة الأساسية: "قطعة"
3. ينقر "إضافة وحدة فرعية"
4. بطاقة جديدة تظهر (#1)
5. يختار الوحدة: "صندوق"
6. يدخل المعامل: 12
7. الملاحظة تظهر فوراً:
   "✅ 1 صندوق = 12 قطعة"
8. يحفظ المنتج
9. البيانات تُحفظ في Firebase:
   subUnits: [{ unitId: "box_id", factor: 12 }]
```

### **عند التعديل:**

```
1. المستخدم ينقر "تعديل" على منتج
2. البيانات تُحمّل من Firebase
3. إذا كان لديه subUnits:
   - يُنشئ بطاقة لكل وحدة فرعية
   - يملأ البيانات
   - يحدث الملاحظة التوضيحية
4. المستخدم يعدّل ويحفظ
5. التحديث يُطبّق في Firebase
```
