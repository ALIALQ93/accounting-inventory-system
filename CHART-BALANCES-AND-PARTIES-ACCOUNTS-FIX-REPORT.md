# 📊 تقرير إصلاحات عرض الأرصدة وخصائص حسابات العملاء/الموردين

## ✅ التحسينات المنفذة:

---

## 1️⃣ عرض الأرصدة في شجرة الحسابات

### ✅ المشكلة السابقة:
- لم تكن الأرصدة تُجمع بشكل صحيح للحسابات الرئيسية
- لم تكن العملة تُعرض بشكل صحيح
- لم يكن هناك مراعاة للعملات المختلفة عند التجميع

### ✅ الحل المنفذ:

#### أ) تحسين تجميع الأرصدة مع مراعاة العملة:
```javascript
// ✅ الآن يتم التحقق من العملة قبل التجميع
const accountCurrency = account.currency || 'IQD';
children.forEach(child => {
    const childBalance = accountBalances[child.id];
    if (childBalance) {
        const childCurrency = childBalance.currency || accountCurrency;
        // ✅ إذا كانت نفس العملة، قم بالجمع مباشرة
        if (childCurrency === accountCurrency) {
            totalDebit += childBalance.debit || 0;
            totalCredit += childBalance.credit || 0;
        } else {
            // ⚠️ إذا كانت عملة مختلفة، لا يتم التجميع (يمكن تحسينها لاحقاً)
            console.warn(`⚠️ Currency mismatch: Parent (${accountCurrency}) vs Child (${childCurrency})`);
        }
    }
});
```

#### ب) عرض العملة في الشجرة والجدول:
- ✅ يتم عرض العملة بجانب كل رصيد
- ✅ التنسيق: `المبلغ + العملة + نوع الرصيد (مدين/دائن)`
- ✅ الألوان:
  - 🔵 أزرق للرصيد المدين
  - 🟢 أخضر للرصيد الدائن

## 2️⃣ خصائص حسابات العملاء/الموردين

### ✅ المشكلة السابقة:
- العملة والطبيعة كانت تُؤخذ من الحساب الرئيسي فقط
- لم يكن هناك مراعاة لبطاقة العميل/المورد

### ✅ الحل المنفذ:

#### أ) تحديث دالة `createSubAccount`:

```javascript
async createSubAccount(
    parentAccountId, 
    partyName, 
    partyCode, 
    partyType = null, 
    partyCurrency = null,  // ✅ جديد
    partyNature = null      // ✅ جديد
) {
    // ✅ تحديد العملة: أولوية لبطاقة العميل/المورد
    const finalCurrency = partyCurrency || parentAccount.currency || 'IQD';
    
    // ✅ تحديد الطبيعة: أولوية لبطاقة العميل/المورد
    let finalNature = partyNature;
    if (!finalNature) {
        if (partyType === 'customer') {
            finalNature = 'debit';    // ✅ العملاء دائماً مدينين
        } else if (partyType === 'supplier') {
            finalNature = 'credit';   // ✅ الموردين دائماً دائنين
        } else {
            finalNature = parentAccount.nature || 'debit';
        }
    }
    
    // ✅ حفظ في البيانات
    const subAccountData = {
        currency: finalCurrency,  // ✅ العملة من بطاقة العميل/المورد
        nature: finalNature,      // ✅ الطبيعة من بطاقة العميل/المورد
        // ... باقي البيانات
    };
}
```

#### ب) أولوية القيم:

| الخاصية | الأولوية 1 | الأولوية 2 | الأولوية 3 |
|---------|-----------|-----------|-----------|
| **العملة** | بطاقة العميل/المورد | الحساب الرئيسي | IQD (افتراضي) |
| **الطبيعة** | بطاقة العميل/المورد | تلقائي حسب النوع | الحساب الرئيسي |

#### ج) القيم التلقائية:

- ✅ **العملاء (customers)**: `nature = 'debit'` (مدين) دائماً
- ✅ **الموردين (suppliers)**: `nature = 'credit'` (دائن) دائماً

## 3️⃣ كيفية الاستخدام

### عند إضافة عميل/مورد جديد:

```javascript
// في نموذج إضافة العميل/المورد:
{
    name: "اسم العميل",
    code: "C001",
    type: "customer",
    accountId: "parent_account_id",
    currency: "USD",        // ✅ يتم أخذه من النموذج
    nature: "debit"         // ✅ يتم أخذه من النموذج (أو تلقائي)
}

// عند إنشاء الحساب:
await createSubAccount(
    data.accountId,
    data.name,
    data.code,
    data.type,
    data.currency,  // ✅ يتم تمرير العملة
    data.nature     // ✅ يتم تمرير الطبيعة
);
```

### النتيجة في شجرة الحسابات:

```javascript
{
    id: "sub_account_123",
    code: "1-2-1.C001",
    name: "اسم العميل",
    currency: "USD",        // ✅ من بطاقة العميل
    nature: "debit",        // ✅ من بطاقة العميل (أو تلقائي)
    isCustomerAccount: true,
    // ... باقي البيانات
```

## 4️⃣ عرض الأرصدة في الشجرة

### ✅ التنسيق الحالي:

```
[المبلغ بالعملة]
[العملة] - [نوع الرصيد]

مثال:
1,250.50
USD - مدين
```

### ✅ الألوان:
- 🔵 **أزرق** للرصيد المدين (`text-primary`)
- 🟢 **أخضر** للرصيد الدائن (`text-success`)

### ✅ التجميع:
- يتم جمع أرصدة الأبناء للحسابات الرئيسية
- يتم التحقق من توحيد العملة قبل التجميع
- إذا كانت العملات مختلفة، لا يتم التجميع (تحذير في Console)

## 5️⃣ ملاحظات مهمة

### ✅ العملة:
- إذا لم تُحدد في بطاقة العميل/المورد، تُؤخذ من الحساب الرئيسي
- إذا لم تُحدد في الحساب الرئيسي، تُستخدم IQD كافتراضي

### ✅ الطبيعة:
- للعملاء: دائماً `debit` (مدين)
- للموردين: دائماً `credit` (دائن)
- يمكن تجاوزها من النموذج

### ✅ التجميع:
- يتم جمع الأرصدة فقط إذا كانت العملات متطابقة
- إذا كانت العملات مختلفة، يتم إظهار تحذير في Console
- يمكن تطوير تحويل العملات لاحقاً
