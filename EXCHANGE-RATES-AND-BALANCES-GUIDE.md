## 🔄 آلية التحويل والحفظ

### 1️⃣ عند حفظ السند (Voucher):

#### أ) في السند:
```javascript
// بيانات العملية في السند:
{
    currency: "USD",              // عملة العملية
    debit: 1000,                  // المبلغ بالعملة الأصلية
    credit: 0,
    exchangeRate: 1500           // سعر الصرف لحظة العملية
}
```

#### ب) عند التحويل للعملة الرئيسية:
```javascript
// التحويل يتم في createGeneralEntry():
const mainCurrency = voucherData.mainCurrency || 'IQD';  // العملة الرئيسية (IQD)

// تحويل المبلغ:
const debitInMainCurrency = await this.convertCurrency(
    entryDebit,        // 1000 USD
    entryCurrency,     // USD
    mainCurrency       // IQD
);  // النتيجة: 1000 * 1500 = 1,500,000 IQD
```

#### ج) حفظ في القيد العام (General Entry):
```javascript
{
    accountId: "account_123",
    debit: 1500000,                // ✅ محول للعملة الرئيسية (IQD)
    credit: 0,
    currency: "IQD",               // ✅ العملة الرئيسية
    originalCurrency: "USD",       // ✅ العملة الأصلية
    exchangeRate: 1500,            // ✅ سعر الصرف المستخدم
    originalDebit: 1000,           // ✅ المبلغ الأصلي
    originalCredit: 0
}
```

## 2️⃣ عند حساب الأرصدة في شجرة الحسابات:

### الكود الحالي:
```javascript
async calculateAndDisplayBalances() {
    // ✅ جلب جميع القيود العامة المرحّلة
    const entriesSnapshot = await db.collection('generalEntries')
        .where('status', '==', 'posted')
        .get();
    
    // ✅ جمع المبالغ المحولة
    entriesSnapshot.forEach(doc => {
        const generalEntry = doc.data();
        generalEntry.entries.forEach(entry => {
            // ✅ المبالغ هنا بالفعل محولة للعملة الرئيسية
            accountBalances[accountId].debit += parseFloat(entry.debit) || 0;
            accountBalances[accountId].credit += parseFloat(entry.credit) || 0;
        });
    });
}
```

## ⚠️ المشكلة الحالية:

### ❌ المشكلة:
الكود الحالي يحول **جميع** العمليات للعملة الرئيسية (IQD) فقط، ولا يراعي:

1. ✅ **عملة الحساب**: إذا كان الحساب بعملة مختلفة (مثل USD)، يتم تخزين القيمة بالـ IQD
2. ✅ **سعر الصرف التاريخي**: كل عملية لها سعر صرف لحظة تنفيذها
3. ✅ **عرض الأرصدة**: تُعرض بالعملة المحددة للحساب، لكن المبالغ محفوظة بالـ IQD

### ✅ الحل الحالي:
- ✅ يتم التحويل للعملة الرئيسية (IQD) باستخدام سعر الصرف لحظة العملية
- ✅ يتم حفظ سعر الصرف الأصلي والسعر الصرف
- ✅ يتم عرض الأرصدة بالعملة المحددة للحساب

## 📋 مثال عملي:

### المثال 1: حساب بعملة USD

#### العملية 1:
```
التاريخ: 01/01/2024
المبلغ: 1,000 USD
سعر الصرف: 1,500 IQD/USD
المبلغ المحول: 1,500,000 IQD
```

#### العملية 2:
```
التاريخ: 15/01/2024
المبلغ: 500 USD
سعر الصرف: 1,520 IQD/USD
المبلغ المحول: 760,000 IQD
```

#### الرصيد النهائي:
```
إجمالي المدين: 1,500,000 + 760,000 = 2,260,000 IQD
الرصيد بالعملة الأصلية: 
  - بالـ IQD: 2,260,000 IQD
  - بالـ USD: 2,260,000 / 1,500 (آخر سعر) = 1,506.67 USD
```

## 🔧 التحسينات المقترحة (اختيارية):

### 1️⃣ حفظ القيم بالعملة الأصلية:
```javascript
// في generalEntry:
{
    debit: 1500000,              // بالعملة الرئيسية (IQD)
    currency: "IQD",             // العملة الرئيسية
    originalDebit: 1000,         // بالعملة الأصلية (USD)
    originalCurrency: "USD",     // العملة الأصلية
    exchangeRate: 1500           // سعر الصرف لحظة العملية
}
```

### 2️⃣ حساب الرصيد بالعملة المحددة للحساب:
```javascript
// عند حساب الرصيد:
const accountCurrency = account.currency || 'IQD';

if (accountCurrency === 'IQD') {
    // استخدم القيمة المحفوظة مباشرة
    balance = accountBalance.debit - accountBalance.credit;
} else {
    // حول من IQD للعملة المحددة
    const currentRate = await getCurrentExchangeRate(accountCurrency);
    balance = (accountBalance.debit - accountBalance.credit) / currentRate;
}
```

### 3️⃣ عرض الأرصدة متعددة العملات:
```javascript
// عرض الرصيد بالعملة المحددة للحساب
const displayBalance = accountBalance / currentExchangeRate;
const displayCurrency = account.currency || 'IQD';
```

## 📊 الخلاصة:

### ✅ ما يعمل حالياً:
1. ✅ **التحويل**: يتم تحويل جميع العمليات للعملة الرئيسية (IQD) باستخدام سعر الصرف لحظة العملية
2. ✅ **الحفظ**: يتم حفظ المبالغ المحولة + المبالغ الأصلية + سعر الصرف
3. ✅ **الحساب**: الأرصدة تُحسب من المبالغ المحولة
4. ✅ **العرض**: تُعرض بالعملة المحددة للحساب (من حساب currency)

### ⚠️ قيود حالية:
1. ⚠️ **كل الحسابات بحسابات بالـ IQD**: حتى لو كانت عملة الحساب مختلفة
2. ⚠️ **سعر الصرف للعرض**: يستخدم آخر سعر صرف لعرض الرصيد بالعملة المحددة
3. ⚠️ **التجميع**: لا يتم تجميع حسابات بعملات مختلفة

### 💡 تحسينات مقترحة:
1. 💡 حفظ القيم بالعملة الأصلية لكل حساب
2. 💡 استخدام سعر الصرف التاريخي للعرض الدقيق
3. 💡 عرض الأرصدة بالعملة الأصلية وبالعملة المحولة
